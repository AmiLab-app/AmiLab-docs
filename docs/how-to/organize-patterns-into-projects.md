# Organize your patterns into projects

A **project** groups related patterns and gives them a shared [gauge](../explanations/gauge-and-dimensions.md) and [color palette](../explanations/the-color-model.md). The **Files** dialog is where you manage all your saved patterns and projects.

You need to be signed in; the dialog manages patterns stored on your account.

## Open the Files dialog

In the toolbar, click **Files**. The **Patterns & Projects** dialog lists each project with its gauge and palette, followed by any ungrouped patterns.

## Work with a pattern

Each pattern row has these actions:

| Action | What it does |
|---|---|
| **Load** | Open the pattern in the editor. |
| Rename (pencil) | Give the pattern a new name. |
| Duplicate (copy) | Make a copy you can edit independently. |
| **Move** | Move the pattern to another project. |
| Delete (trash) | Remove the pattern. Asks for confirmation. |

## Work with projects

- **Create a project** — click **New project** and set its name, gauge, and palette.
- **Add a pattern to a project** — use the **New pattern** button inside a project's section, or **Move** an existing pattern into it.
- **Edit a project** — click the pencil on the project's header to change its name, gauge, or palette. The gauge and palette apply to every pattern in the project.
- **Delete a project** — click the trash on the project's header.

## Ungrouped patterns

Patterns created without a project appear under **Ungrouped**. They behave the same way, but carry their own gauge and color instead of sharing a project's. Move one into a project at any time with **Move**.
