# AmiLabがパーツを表現する仕組み

AmiLabの動作を理解するうえで最も重要なのは、**編み図は編み目のグラフであり、画面に表示される段はそのグラフから求められる**という考え方です。編み目の接続を理解すると、増目、減目、編集の結果を予測できるようになります。

## 編み目と土台の目

各編み目には、ここで重要になる3つの情報があります。

- **種類** — `細`、`中長`、`長`、`長々`、`鎖`のいずれか（[編み目の種類](../reference/stitch-types.md)を参照）
- **土台** — その目を編み入れる、1段下の目
- 任意の**色**

中心になるのは土台です。編み目は、自分が載っている下の目を参照します。

- **土台なし** — その目からパーツが始まります。わの作り目の中心や作り目の鎖編みが該当します。
- **土台が1目** — 1段下の1目に編み入れる通常の編み目です。
- **土台が2目以上** — 複数の目を一度に編む**減目**です。段の目数が減ります。

**増目**はこの逆ですが、専用の記録は必要ありません。同じ段の2目が同じ土台を共有していれば、1つの目に2目編み入れた増目だと判断できます。AmiLabはグラフの形から増目を認識します。

## 段は保存せず、接続から求める

AmiLabは「3段目」という情報を直接保存しません。編み図が変わるたびに、土台との接続から段を計算します。

- 土台を持たない作り目が最初の段になります。
- 編み目は、最も深い土台の1段上に属します。

そのため、土台を変更したり、目を分割したり、削除したりしても、編み目グリッドは接続関係と一致した状態を保ちます。保存済みの配置を修正するのではなく、接続から段を計算し直すためです。

## この仕組みが操作に与える影響

- **増目と減目は接続関係で決まります。** パーツの形は、同じ土台を共有する目の数を変えて作ります。別の「増目」フラグを切り替えるわけではありません。
- **編集結果が一貫します。** 段の目数、文章の編み図、3Dプレビューは、すべて同じグラフを参照します。
- **3Dプレビューも同じグラフから作られます。** 土台の接続により、どの目がどの目の上にあるかを判断し、各目を空間に配置します。
